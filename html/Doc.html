JS web app for ev3dev

WS is a web server with web socket functions and build in javascript engine for ev3dev linux distro.

The ide is a development platform in a web application.

Your projects is build of to javascript files, edited directly in the web app, one script to run on the brick and one script to extend the web app. And there is serverel examples on that in the installation. In your brick script , you declare the function loop, a fall throu function.

var a=0;

function loop() {

if(a!=Motor.outA.read(“position”)) {

a=Motor.outA.read(“position”);

Msg.log(“Motor position: ” + a);

}

}

In this example we check if motor position has changed, and report it if it has.

Here we see two of the build in objects, Msg and Motors, we have the following obj available in the brick script environment:

Motors – Contains motor objects, that contains basic controls for each motor.

Methods:

read(property);

write(property,value);

Example:

var motorA = Motors.outA;

motorA.write(“command”,”reset”);

Sensors

Methods:

read(property);

write(property,value);

Example:

var t = Sensor.in1.read(“value0”);

Msg.log(“Sensor 1 value: ” + t);

Msg

Methods:

log(msg);

send(channel,msg);



In init() you create all your objects and in loop you use them.

All ports are enummerated at startup by the Ports object that are global stored in “Ports *ports”. There is a special “port”, Messenger, it is used to send messages thrue the web server to

the web page(see test-radar). It is manuelly created to bind to a channel and must be done in init.

To use the ports workers are made for all types of devices:

Touch for touch sensor.

ColColor for color sensor in COL-COLOR mode.

Prox for ir sensor in IR-PROX mode

Remote for ir sensor in IR-REM-A mode.
